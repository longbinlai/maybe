## 2. 系统设计：利用现有 Memory 系统

### 架构

```
宏观新闻（每日）
    ↓ collect_macro_info.py 收集
    ↓ OpenClaw AI 翻译和总结
    ↓ 自动写入 MEMORY.md（结构化格式）
    ↓ OpenClaw 自动索引
    ↓ 语义搜索可用

家庭决策（每次）
    ↓ 用户在飞书对话中说明决策
    ↓ OpenClaw AI 提取结构化信息
    ↓ 自动写入 MEMORY.md
    ↓ 建立决策历史

投资分析（按需）
    ↓ 用户提问："美联储加息对我们有什么影响？"
    ↓ OpenClaw AI 查询 MEMORY.md（语义搜索）
    ↓ 找到相关的历史事件和决策
    ↓ 生成综合分析报告
```

### 关键：让 AI 自动维护 MEMORY.md

**不需要**：
- ❌ 自己实现 SQLite 数据库
- ❌ 自己实现向量搜索
- ❌ 自己实现文件索引

**需要**：
- ✅ 让 AI 知道如何写入 MEMORY.md
- ✅ 定义清晰的格式规范
- ✅ 让 AI 知道何时写入
- ✅ 让 AI 知道如何查询

---

## 3. 实现方案

### 3.1 更新 SKILL.md：指导 AI 维护 MEMORY.md

在 `macro-info-collector/SKILL.md` 中添加：

```markdown
## Memory 管理规则

### 每日宏观新闻处理

当收集到宏观新闻时，**必须**：

1. **提取关键事件**（影响 > 阈值的事件）
2. **写入 MEMORY.md** 的 `## Market Events` 部分
3. **使用标准格式**：

```markdown
### YYYY-MM-DD: [事件标题]
- **类别**: [policy/inflation/employment/market]
- **市场**: [涉及的市场]
- **关键数据**: [具体数字]
- **情绪**: [hawkish/dovish/neutral]
- **影响板块**: [bonds/stocks/gold/real_estate]
- **相关性**: [high/medium/low]
- **来源**: [信息源]
- **链接**: [URL]
```

### 家庭决策记录

当用户做出投资决策时，**必须**：

1. **提取决策信息**
2. **写入 MEMORY.md** 的 `## Decision Log` 部分
3. **使用标准格式**：

```markdown
### YYYY-MM-DD: [决策标题]
- **行动**: [buy/sell/rebalance/hold]
- **账户**: [账户名]
- **证券**: [代码]
- **数量**: [股数]
- **价格**: [价格]
- **总价值**: [金额]
- **理由**: [为什么]
- **市场背景**: [当时的宏观环境]
- **预期结果**: [期望发生什么]
- **信心指数**: [1-10]
- **状态**: [active/closed/reviewing]
```

### 查询历史

当用户询问历史问题时，**应该**：

1. **使用语义搜索**：`openclaw memory search "关键词"`
2. **分析相关记录**
3. **提供历史参考**

示例：
```
用户：美联储上次加息时我们做了什么？

AI 流程：
1. 搜索 MEMORY.md 中包含"美联储"和"加息"的记录
2. 找到相关的 Market Events 和 Decision Log
3. 分析当时的决策和结果
4. 提供综合回答
```
```

### 3.2 创建自动写入的 Cron 任务

**每日宏观事件提取**：

```bash
openclaw cron add \
  --name "macro-event-extraction" \
  --cron "0 22 * * *" \
  --message "请执行以下步骤：
1. 收集今天的宏观新闻
2. 提取重要事件（相关性 >= medium）
3. 将事件写入 MEMORY.md 的 '## Market Events' 部分
4. 使用标准格式（参考 SKILL.md）
5. 不要重复写入已存在的事件"
```

### 3.3 创建决策记录的 Skill

**family-decision-logger Skill**：

```markdown
---
name: family-decision-logger
description: "记录家庭投资决策到 MEMORY.md"
---

## 使用方式

当用户在对话中提到投资决策时：

1. **提取决策信息**：
   - 行动（买入/卖出/调整/持有）
   - 账户、证券、数量、价格
   - 理由和背景

2. **查询当前宏观环境**：
   - 使用 `openclaw memory search` 查找最近的市场事件
   - 记录当时的宏观背景

3. **写入 MEMORY.md**：
   - 追加到 `## Decision Log` 部分
   - 使用标准格式
   - 包含时间戳

4. **确认记录**：
   - 告诉用户已记录
   - 提供记录摘要

## 示例对话

用户：今天把长桥R账户的苹果股票卖了，150美元卖的

AI：
1. 提取信息：
   - 行动：卖出
   - 账户：长桥R
   - 证券：AAPL
   - 价格：$150

2. 查询背景：
   - 搜索"美联储"、"科技股"等关键词
   - 找到最近的市场事件

3. 写入记录：
   ```
   ### 2026-05-24: 卖出苹果股票
   - **行动**: 卖出
   - **账户**: 长桥R
   - **证券**: AAPL
   - **价格**: $150/股
   - **理由**: [询问用户]
   - **市场背景**: 美联储鹰派，科技股承压
   - **状态**: closed
   ```

4. 确认：
   "已记录你的决策：卖出 AAPL @ $150。
   原因是什么？（可选）"
```

### 3.4 创建投资分析的 Skill

**family-investment-analyzer Skill**：

```markdown
---
name: family-investment-analyzer
description: "基于历史记录分析家庭投资决策"
---

## 使用方式

当用户询问投资相关问题时：

1. **解析问题**：
   - 识别查询类型（历史对比/趋势分析/决策评估）
   - 提取关键词

2. **搜索历史记录**：
   - 使用 `openclaw memory search` 查询相关事件
   - 分析 Market Events 和 Decision Log

3. **生成分析报告**：
   - 对比历史案例
   - 评估决策效果
   - 提供建议

## 示例对话

用户：美联储加息时我们应该怎么做？

AI：
1. 搜索历史记录：
   ```bash
   openclaw memory search "美联储 加息"
   openclaw memory search "Fed rate hike"
   ```

2. 找到相关记录：
   - 2024-03-15: 美联储加息，我们减持债券
   - 2023-07-20: 美联储加息，我们未调整

3. 分析结果：
   ```
   历史经验：
   
   2024-03-15（加息50bp）：
   - 我们的决策：减持10%债券，增持5%黄金
   - 结果：+2.3%（30天内）
   - 原因：加息导致债券下跌，黄金上涨
   
   2023-07-20（加息25bp）：
   - 我们的决策：未调整
   - 结果：-1.8%（30天内）
   - 原因：债券下跌，我们持有过多债券
   
   建议：
   根据历史经验，美联储加息时：
   - 减持长久期债券（预期下跌2-5%）
   - 增持黄金（预期上涨3-8%）
   - 保持现金（提供灵活性）
   ```

4. 提供当前建议：
   "基于历史经验，建议：
   - 减持债券至15%（当前25%）
   - 增持黄金至20%（当前15%）
   - 预期影响：+1.5% to +3%（30天内）"
```

## 4. 实际使用流程

### 每日流程

```
早上 8:00（Cron 任务）
├── 收集宏观新闻
├── 翻译和总结
├── 提取重要事件
├── 写入 MEMORY.md
└── 推送到飞书

晚上 22:00（Cron 任务）
├── 回顾当天事件
├── 提取遗漏的重要事件
├── 补充写入 MEMORY.md
└── 更新索引
```

### 决策记录流程

```
用户："今天买了100股腾讯"
├── AI 提取决策信息
├── 查询当前宏观环境
├── 写入 MEMORY.md
├── 确认记录
└── 更新索引
```

### 分析查询流程

```
用户："上次美联储加息我们做了什么？"
├── AI 解析问题
├── 搜索 MEMORY.md（语义搜索）
├── 找到相关记录
├── 分析历史案例
├── 生成综合报告
└── 提供建议
```
