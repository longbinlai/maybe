### ✅ 数据流是否正确

| 流程 | 数据来源 | 是否正确 |
|------|---------|---------|
| 决策记录 | Maybe CLI + 日期文件 | ✅ |

| 月度分析 | 周度回顾 + Maybe snapshot | ✅ |

### ✅ 是否需要额外追踪表

**答案**: 不需要

**原因**:
- Maybe 已经有所有交易数据（ticker、数量、价格）
- 记忆文件已经有理由和预期
- 周度/月度回顾时，Agent 可以：
  1. 读日期文件 → 找到决策
  2. 查 Maybe 当前价格 → 计算盈亏
  3. 对比预期和实际 → 写评估

## 结论

### 三层架构设计是合理的

1. **Maybe** 负责事实数据（交易、持仓、价格）
2. **记忆** 负责上下文数据（理由、预期、市场背景）
3. **回顾** 负责评估数据（对不对、学到什么）

### 不需要额外追踪表

- Maybe 已经有所有交易数据
- 记忆文件已经有理由和预期
- Agent 可以在回顾时动态关联两者

### 更新频率合理

| 层 | 更新频率 | 是否合理 |
|----|---------|---------|
| Maybe | 每次交易时 | ✅（稀疏事件） |
| 记忆 | 每次决策时 | ✅（稀疏事件） |
| 周度回顾 | 每周日 | ✅（定期） |
| 月度分析 | 每月1日 | ✅（定期） |
