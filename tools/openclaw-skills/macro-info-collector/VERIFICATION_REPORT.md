## 三层架构验证

### 数据流

```
用户: "买入100股腾讯 @ $380"
  ↓
Agent 执行:
  ├─→ Maybe CLI（事实层）
  │   maybe holding add --account 长桥R --ticker 0700.HK --qty 100
  │   记录: ticker, 数量, 价格, 日期
  │
  └─→ 记忆文件（上下文层）
      memory/2026-05-20.md
      记录: 理由, 预期, 市场背景
  ↓
周度回顾（评估层）:
  ├─→ 读取 memory/2026-05-20.md（找到决策）
  ├─→ 查询 maybe holdings（获取当前价格）
  ├─→ 对比预期和实际（计算盈亏）
  └─→ 写入 memory/2026-05-25-weekly.md（评估 + 建议）
  ↓
更新 MEMORY.md:
  └─→ 提取投资洞察（从周度回顾）
```

## 关键发现

### 1. 不需要额外追踪表

**原因**:
- Maybe 已经有所有交易数据（ticker、数量、价格）
- 记忆文件已经有理由和预期
- Agent 可以在回顾时动态关联两者

**示例**:
```
决策记录（memory/2026-05-20.md）:
- ticker: 0700.HK
- 价格: $380
- 预期: +15% in 90d

Maybe holdings（实时查询）:
- ticker: 0700.HK
- 当前价格: $392
- 盈亏: +3.16%

Agent 动态关联:
- 读取决策记录 → 得到预期
- 查询 Maybe → 得到当前价格
- 对比 → 计算评估
```

---

### 2. 更新频率合理

| 层 | 更新频率 | 是否合理 |
|----|---------|---------|
| Maybe | 每次交易时 | ✅（稀疏事件） |
| 记忆 | 每次决策时 | ✅（稀疏事件） |
| 周度回顾 | 每周日 | ✅（定期） |
| 月度分析 | 每月1日 | ✅（定期） |

**结论**: 大多数日子是零决策，不需要每日更新。

---

### 3. 语义搜索足够用

**测试结果**:
- 搜索 "腾讯 买入" → 找到4个相关片段
- 最高分: 0.430（投资洞察）
- 最低分: 0.394（操作建议）

**结论**: 语义搜索可以正确找到相关记录，不需要结构化查询。

## 结论

### ✅ 三层架构设计是合理的

1. **Maybe** 负责事实数据（交易、持仓、价格）
2. **记忆** 负责上下文数据（理由、预期、市场背景）
3. **回顾** 负责评估数据（对不对、学到什么）

### ✅ 不需要额外追踪表

- Maybe 已经有所有交易数据
- 记忆文件已经有理由和预期
- Agent 可以在回顾时动态关联两者

### ✅ 更新频率合理

- 决策是稀疏事件（每月3-5次）
- 大多数日子是零决策
- 周度/月度回顾足够

### ✅ 语义搜索足够用

- 可以正确找到相关记录
- 不需要结构化查询
- 满足用户的真实问题
