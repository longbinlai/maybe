# 家庭投资决策管理

管理家庭投资决策的记忆系统。与 Maybe Finance（事实数据）配合使用。

## 三层架构

```
Maybe（事实层）               记忆（上下文层）              回顾（评估层）
─────────────────          ─────────────────          ─────────────────
持仓、交易、价格              每日事件 + 决策理由           周度回顾
账户、汇率、净值              市场事件 + 投资洞察           月度分析
                             ↓ 记录时写入 ↓               ↓ 回顾时写入 ↓
                           memory/YYYY-MM-DD.md         memory/YYYY-MM-DD-weekly.md
                                                        memory/YYYY-MM-monthly.md

全部通过 memory_search 语义搜索
```

| 层 | 回答什么 | 更新频率 | 由谁写入 |
|---|---|---|---|
| Maybe | 事实：持有什么、值多少 | 实时（每次交易） | maybe CLI |
| 记忆 | 上下文：为什么、预期什么 | 事件驱动（有决策才写） | Agent（对话中） |
| 回顾 | 评估：对不对、学到什么 | 周/月定期 | Agent（cron 任务） |

## When to Use

Use when:

- 用户做出投资决策（买入/卖出/调仓/还贷）
- 用户询问历史决策（"上次买腾讯赚了吗？"）
- 执行周度回顾（每周日 20:00）
- 执行月度分析（每月 1 日 20:00）
- 需要查询投资组合表现和决策质量

## When NOT to Use

Do not use when:

- 只是查询市场数据 → 用 `macro-info-collector` 或 `yfinance` skill
- 只是查询持仓 → 用 `maybe` skill
- 需要更新账户余额 → 用 `maybe-reconcile` skill

## 投资决策记录流程

当用户做出投资决策时，**同时在两个系统中记录**：

### 第1步：调用 Maybe CLI 记录交易（事实层）

Maybe 负责记录"什么"：ticker、数量、价格、日期、账户

```bash
# 示例：买入100股腾讯
maybe holding add --account 长桥R --ticker 0700.HK --qty 100

# 示例：卖出全部苹果股票
maybe holding remove --account 长桥R --ticker AAPL

# 示例：调整债券持仓
maybe holding add --account 长桥R --ticker TLT --qty 50
```

### 第2步：写入日期文件（上下文层）

记忆负责记录"为什么"和"预期什么"：

- 写入 `memory/YYYY-MM-DD.md` 的 `## 决策记录` 部分
- 使用 MEMORY.md 中定义的标准模板格式
- **必须包含**：
  - 理由（为什么做这个决策）
  - 预期结果（期望发生什么，如"+5% in 30d"）
  - 信心指数（1-10）
  - 市场背景（通过 `memory_search` 查询当时的市场事件）

**示例**：
```markdown
### 2026-05-25: 买入腾讯
- **行动**: buy
- **账户**: 长桥R
- **证券**: 0700.HK
- **数量**: 100
- **价格**: $380
- **总价值**: $38,000
- **理由**: AI业务增长强劲，估值合理
- **预期结果**: +15% in 90d
- **信心指数**: 7
- **市场背景**: 恒生指数连续3天上涨，科技板块表现强劲
- **状态**: active
```

### 第3步：确认记录完成

向用户确认两件事都已完成：
- ✅ Maybe 已记录交易
- ✅ 记忆已记录理由和预期

## 周度回顾流程（每周日 20:00）

**信息来源**（不需要额外追踪表）：

| 来源 | 提供什么 |
|------|---------|
| `memory/` 本周日期文件 | 本周做了哪些决策（理由、预期） |
| `maybe trades --json` | 本周实际交易（价格、数量） |
| `maybe holdings --json` | 当前持仓和盈亏 |
| `memory_search "本周日期范围"` | 本周市场事件（宏观环境） |

**执行流程**：

1. **读取本周日期文件**
   ```bash
   # 读取本周所有日期文件
   cat memory/2026-05-20.md memory/2026-05-21.md ... memory/2026-05-25.md
   ```

2. **查询 Maybe 当前状态**
   ```bash
   maybe holdings --json  # 当前持仓和盈亏
   maybe trades --json    # 本周交易记录
   ```

3. **评估每个决策**
   - 对比"预期结果"和"实际盈亏"
   - 写评估（correct/incorrect/partial）
   - 写教训

4. **写入周度回顾文件**
   - 文件路径：`memory/YYYY-MM-DD-weekly.md`
   - 使用 MEMORY.md 中的周度回顾模板
   - **给出具体操作建议**

5. **更新 MEMORY.md**
   - 如果有新的投资洞察，追加到「投资洞察」部分

**示例评估**：
```markdown
### 决策评估

#### 2026-05-20: 买入腾讯
- **预期**: +15% in 90d
- **实际**: +3.2% (当前 $392)
- **评估**: partial（方向正确，但幅度不及预期）
- **教训**: AI概念股波动比预期小，需要更保守的预期

#### 2026-05-22: 卖出苹果
- **预期**: 避险，预计下跌5%
- **实际**: -2.1% (卖出价 $150，当前 $147)
- **评估**: correct（成功避险）
- **教训**: 美联储鹰派信号时减持科技股是有效的
```

## 月度分析流程（每月 1 日 20:00）

**信息来源**：

| 来源 | 提供什么 |
|------|---------|
| `memory/` 本月周度回顾文件 | 本月所有决策评估 |
| `maybe snapshot --json` | 当前完整持仓 |
| `maybe trades --json` | 本月所有交易 |

**执行流程**：

1. **读取本月周度回顾**
   ```bash
   cat memory/2026-05-05-weekly.md memory/2026-05-12-weekly.md ...
   ```

2. **分析长期趋势**
   - 统计决策成功率
   - 识别模式（哪些类型的决策更成功）

3. **给出策略调整建议**
   - 基于历史教训
   - 给出资产配置调整建议

4. **写入月度分析文件**
   - 文件路径：`memory/YYYY-MM-monthly.md`
   - 使用 MEMORY.md 中的月度分析模板

5. **更新 MEMORY.md**
   - 更新「当前投资组合」部分
   - 追加新的「投资洞察」

## 查询历史记录

- 使用 `memory_search "关键词"` 搜索所有文件
- 或直接读取对应日期文件 `memory/YYYY-MM-DD.md`

**示例查询**：

用户问："上次买腾讯赚了吗？"

1. `memory_search "腾讯"` → 找到决策记录
2. 从决策记录中提取: 买入价 $380，预期 +15% in 90d
3. `maybe holdings --ticker 0700.HK` → 当前价格 $392
4. 计算: ($392 - $380) / $380 = +3.16%
5. 对比: 实际 +3.16% vs 预期 +15% in 90d

回答：
```
上次买腾讯（2026-05-20）：
- 买入价: $380
- 当前价: $392
- 收益: +3.16% (5天)
- 预期: +15% in 90d
- 评估: 方向正确，速度是预期的3.78倍
```

## 关键原则

1. **不需要额外追踪表** — Maybe 已经有所有交易数据，记忆文件有理由和预期，Agent 可以在回顾时动态关联两者
2. **更新频率合理** — 决策是稀疏事件（每月3-5次），大多数日子是零决策
3. **语义搜索足够用** — 可以正确找到相关记录，不需要结构化查询
4. **Maybe 和记忆分工明确** — Maybe 回答"什么"，记忆回答"为什么"和"所以呢"
