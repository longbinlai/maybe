### 能力 2：修改家庭投资明细（Maybe）
**触发方式**：用户明确要求修改
**执行者**：OpenClaw AI
**工具**：Maybe CLI + OpenClaw Skill

**示例对话**：
```
用户：我要买入 100 股腾讯

AI 执行：
1. 确认信息：
   - 买入：腾讯 (0700.HK)
   - 数量：100 股

2. 询问确认：
   "确认买入 100 股腾讯 @ $380，总计 $38,000？"
   
3. 用户确认后执行：
   maybe holding add --account "长桥R" --ticker "0700.HK" --qty 100
   
4. 记录决策到 MEMORY.md：
   ### 2026-05-24: 买入腾讯
   - **行动**: 买入
   - **账户**: 长桥R
   - **证券**: 0700.HK (腾讯)
   - **数量**: 100 股
   - **价格**: $380/股
   - **总价值**: $38,000
   - **理由**: [询问用户]
   - **市场背景**: [从 MEMORY.md 查询最近宏观事件]
   - **状态**: active
   
5. 更新 MEMORY.md
6. 确认："已记录买入决策"
```

### 能力 3：每日获取宏观经济信号（macro-info-collector）
**触发方式**：Cron 定时任务（每日 8:00）
**执行者**：OpenClaw AI + Cron
**工具**：collect_macro_info.py + OpenClaw AI

**执行流程**：

#### 早上 8:00（每日早报 - 仅信息收集，不给投资建议）
```
1. 收集宏观数据：
   python3 collect_macro_info.py

2. AI 翻译和总结

3. 提取重要事件（相关性 >= medium）

4. 查询当前投资组合：
   maybe snapshot --json

5. 交叉分析（仅展示影响，不给建议）：
   - 哪些持仓受到影响？
   - 影响程度如何？（高/中/低）
   - 历史类似情况发生了什么？（客观事实）

6. 生成报告：
   📈 宏观经济早报 - 2026-05-24

   【重要事件】
   🔴 美联储加息 50bp（超出预期）
   🟡 中国 PMI 数据：50.2（略高于荣枯线）
   🟢 欧洲通胀下降：2.8% → 2.5%

   【你的持仓影响】
   ⚠️ TLT（债券ETF）：可能下跌 2-3%
   ⚠️ VNQ（房地产ETF）：可能下跌 1-2%

   【历史参考】
   • 2024-03-15（加息50bp）：债券下跌3.2%，黄金上涨5.1%
   • 2023-07-20（加息25bp）：债券下跌1.8%，黄金上涨2.3%

   周日晚上的周度回顾将给出具体操作建议。

7. 写入 MEMORY.md（市场事件部分）

8. 推送到飞书

**注意**：早报只提供客观信息和历史参考，**不给出操作建议**。投资建议在周度回顾中给出。
```

## 核心：投资建议生成算法

### 触发条件

**主动触发（AI 主动建议）**：
1. **周度回顾**（每周日 20:00）：
   - 基于本周宏观新闻和市场事件
   - 基于本周用户操作和对话记录
   - 基于持仓变化和风险暴露
   - 给出下周操作建议

2. **月度深度分析**（每月 1 日 9:00）：
   - 基于本月 4 次周度回顾的总结
   - 基于关键记忆和信号的长期趋势
   - 基于决策质量评估和历史教训
   - 给出下月策略调整建议

**被动触发（用户询问）**：
1. 用户问："我们应该调仓吗？"
2. 用户问："当前环境适合买/卖 XX 吗？"
3. 用户问："上次类似情况我们做了什么？"

### 建议生成流程

```
步骤 1：识别触发条件
├── 是宏观事件？→ 提取事件类型、影响板块
├── 是配置漂移？→ 计算漂移程度
└── 是定期审查？→ 全面分析

步骤 2：查询当前投资组合
maybe snapshot --json
├── 获取所有持仓
├── 计算当前配置比例
└── 识别受影响的持仓

步骤 3：查询历史记录
openclaw memory search "相关关键词"
├── 查找类似历史事件
├── 查找过去的决策
└── 评估历史决策结果

步骤 4：生成建议
├── 列出所有可能的操作
├── 评估每个操作的风险/收益
├── 排序（按优先级）
└── 生成具体建议

步骤 5：呈现给用户
├── 解释为什么建议
├── 提供历史参考
├── 给出具体操作步骤
└── 询问用户是否执行
```

### 具体建议类型

#### 类型 1：宏观事件驱动

**触发**：美联储加息 50bp

**分析流程**：
```
1. 事件影响分析：
   - 加息 → 债券价格下跌
   - 加息 → 成长股承压
   - 加息 → 美元走强
   - 加息 → 黄金上涨（避险）

2. 查询当前持仓：
   - 债券：25%（TLT 15%, AGG 10%）
   - 成长股：20%（TSLA 6.5%, NVDA 5%）
   - 黄金：15%（GLD 15%）

3. 查询历史：
   - 2024-03-15（加息 50bp）：减持债券 → +2.3%
   - 2023-07-20（加息 25bp）：未调整 → -1.8%

4. 生成建议：
   
   🔴 高优先级：减持债券
   
   原因：
   • 美联储加息 50bp，超出预期
   • 债券价格预计下跌 3-5%
   • 当前债券占比 25%，风险较高
   
   历史参考：
   • 2024-03-15（类似情况）：减持债券 → +2.3%
   • 2023-07-20（未调整）：-1.8%
   
   建议操作：
   • 减持 TLT 从 15% 降至 10%（卖出约 $61,728）
   • 增持 GLD 从 15% 升至 20%（买入约 $61,728）
   
   预期影响：
   • 避免损失：+1.5% to +3%（vs 不调整）
   • 时间框架：30天内
   
   要执行这个调整吗？
```

#### 类型 2：配置漂移驱动

**触发**：股票占比从 60% 升至 68%（上涨导致）

**分析流程**：
```
1. 漂移分析：
   - 目标配置：股票 60%
   - 当前配置：股票 68%
   - 漂移程度：+8%（超过 5% 阈值）

2. 原因分析：
   - 科技股上涨 15%（AAPL, GOOGL）
   - 债券下跌 3%
   - 黄金持平

3. 风险评估：
   - 集中风险：科技股占比 25%
   - 估值风险：科技股 PE 处于历史高位
   - 波动风险：科技股波动率上升

4. 生成建议：
   
   ⚠️ 中优先级：再平衡投资组合
   
   原因：
   • 股票占比从 60% 升至 68%（+8%）
   • 科技股集中度过高（25%）
   • 估值处于历史高位
   
   建议操作：
   • 减持科技股 5%（卖出约 $61,728）
     - AAPL: 减持 2%
     - GOOGL: 减持 3%
   • 增持债券 3%（买入约 $37,037）
   • 增持黄金 2%（买入约 $24,691）
   
   预期影响：
   • 降低风险：减少 8% 的股票敞口
   • 改善配置：回到目标配置
   • 时间框架：1-2周内完成
   
   要执行再平衡吗？
```

#### 类型 3：定期审查驱动

**触发**：每周日 20:00（周度回顾）

**分析流程**：
```
1. 本周表现回顾：
   - 投资组合：+1.2%
   - 基准（60/40）：+0.8%
   - 超额收益：+0.4%

2. 本周重要事件：
   - 5/20：美联储讲话（中性）
   - 5/22：CPI 数据（低于预期）
   - 5/24：科技股财报（超预期）

3. 本周决策回顾：
   - 5/21：买入腾讯（+3.2%）✓ 正确
   - 5/23：减持 TLT（-1.5%）✗ 错误
   
4. 下周展望：
   - 5/28：PCE 数据（重要）
   - 5/30：GDP 修正值
   - 6/1：PMI 数据

5. 生成建议：
   
   📊 周度回顾 - 2026-05-24
   
   【本周表现】
   • 投资组合：+1.2%（vs 基准 +0.8%）
   • 超额收益：+0.4%
   
   【决策评估】
   ✅ 买入腾讯：+3.2%（正确决策）
   ❌ 减持 TLT：-1.5%（错误决策）
   
   【下周建议】
   ⚠️ 关注 PCE 数据（5/28）
   • 如果通胀高于预期：减持债券
   • 如果通胀低于预期：增持股票
   
   建议：
   • 保持当前配置
   • 等待 PCE 数据后再调整
   • 准备现金（5%）用于机会
```

## 记忆管理策略

### 记忆结构

**MEMORY.md 包含 4 个主要部分**：

```markdown
# 家庭投资记录

## 📋 AI 使用说明
[指导 AI 如何记录和查询]

## 决策记录
[所有投资决策的历史]

## 市场事件
[重要的宏观经济事件]

## 投资组合
[当前配置和目标配置]

## 投资洞察
[从历史中总结的规律]
```

### 记忆更新频率

| 记忆类型 | 更新频率 | 触发条件 | 示例 |
|---------|---------|---------|------|
| 市场事件 | 每日 22:00 | Cron 任务 | "美联储加息 50bp" |
| 决策记录 | 实时 | 用户提到决策 | "买入 100 股腾讯" |
| 投资组合 | 每日 8:00 | Cron 任务 | 当前配置快照 |
| 投资洞察 | 每周日 20:00 | 周度回顾 | "加息时减持债券效果好" |

### 记忆晋升机制

**短期记忆 → 长期记忆**：

```
短期记忆（最近 7 天）：
- 2026-05-20: 美联储讲话（中性）
- 2026-05-22: CPI 低于预期
- 2026-05-24: 买入腾讯

↓ 每周日 20:00（周度回顾）

评估重要性：
- 事件影响程度
- 决策结果好坏
- 是否可重复的规律

↓ 如果重要（评分 > 7/10）

长期记忆（永久保存）：
- 2026-05: 通胀下降时科技股上涨
- 2026-05: 腾讯 AI 概念持续强势
```

### 记忆查询策略

**AI 查询 MEMORY.md 的策略**：

```
用户问："美联储加息时我们应该怎么做？"

AI 查询流程：
1. 语义搜索："美联储 加息"
   → 找到相关的市场事件
   
2. 语义搜索："加息 决策"
   → 找到过去的相关决策
   
3. 语义搜索："加息 结果"
   → 找到决策的结果
   
4. 综合分析：
   - 2024-03-15：加息 50bp → 减持债券 → +2.3%
   - 2023-07-20：加息 25bp → 未调整 → -1.8%
   - 2022-11-10：加息 75bp → 增持黄金 → +3.1%
   
5. 生成建议：
   "根据历史经验，美联储加息时：
   • 减持债券效果好（2/3 次成功）
   • 增持黄金效果好（1/1 次成功）
   • 不调整效果差（0/1 次成功）
   
   建议：减持债券 5-10%，增持黄金 5%"
```

## Cron 任务配置

### 任务 1：每日早报（每天早上 8:00 - 仅信息收集）
```bash
openclaw cron add \
  --name "daily-morning-report" \
  --schedule "0 8 * * *" \
  --message "执行每日早报流程（仅信息收集，不给投资建议）：
1. 收集过去 24 小时的宏观新闻
2. 查询当前投资组合（maybe snapshot）
3. 交叉分析：哪些持仓受影响？（仅展示影响程度）
4. 查询历史记录（MEMORY.md）中的类似事件（仅展示历史事实）
5. 写入 MEMORY.md（市场事件）
6. 推送到飞书（不包含操作建议）"
```

### 任务 2：周度回顾（每周日晚上 20:00 - 给出投资建议）
```bash
openclaw cron add \
  --name "weekly-review" \
  --schedule "0 20 * * 0" \
  --message "执行周度回顾流程（给出投资建议）：
1. 本周宏观新闻汇总（从 MEMORY.md 提取）
2. 本周用户操作和对话记录（从 MEMORY.md 提取）
3. 持仓变化和风险暴露（maybe snapshot）
4. 本周决策评估（哪些正确/错误）
5. 生成投资建议（基于本周新闻、操作和历史经验）
6. 记忆晋升（评估短期记忆，晋升重要的）
7. 下周展望（重要事件预告）
8. 生成周度报告并推送"
```

## 总结

这个系统的关键特点：

1. **信息驱动**：每日早报提供客观信息，不给建议
2. **定期建议**：周度回顾给操作建议，月度分析给策略建议
3. **可追溯**：所有决策和结果都记录在 MEMORY.md
4. **可学习**：从历史中学习，改进未来决策
5. **可操作**：给出具体建议，用户一键执行

**核心价值**：
- 每日早报：**发生了什么**（客观信息）
- 周度回顾：**你应该怎么做**（操作建议）
- 月度分析：**如何调整策略**（长期建议）
